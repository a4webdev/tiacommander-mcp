# Contributing to TiaCommander

Thank you for your interest in TiaCommander!

## Current Status

TiaCommander is currently in **early release**. This repository serves as the
public distribution channel for releases, documentation, and issue tracking.
Source code is maintained in a private repository.

**We are not accepting pull requests at this time.**

## How You Can Help

### Report Bugs

Found a bug? Please [open an issue](https://github.com/a4webdev/tiacommander-mcp/issues/new?template=bug_report.md)
with a clear description, steps to reproduce, and your environment details
(TIA Portal version, Windows version, MCP client).

### Request Features

Have an idea? [Submit a feature request](https://github.com/a4webdev/tiacommander-mcp/issues/new?template=feature_request.md)
describing the problem you're solving and your proposed solution.

### Share Feedback

Use the built-in feedback form in TiaCommander Manager, or reach out at
**info@tiacommander.com**.

## Future Opportunities

We are building a team around TiaCommander and the broader vision of
AI-powered industrial automation. If you have experience in:

- **Siemens PLC programming** (TIA Portal, S7-1200/1500, SCL, LAD, FBD)
- **Industrial automation protocols** (OPC UA, S7Comm, Modbus, PROFINET)
- **.NET / VB.NET development**
- **MCP (Model Context Protocol) server development**

...and you are interested in contributing or joining the team, we would love
to hear from you. Reach out at **info@tiacommander.com** with a brief
introduction and your background.

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md).
By participating, you agree to uphold this code.
